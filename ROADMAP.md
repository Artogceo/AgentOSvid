# VidClaw → AC Integration roadmap

Дорожная карта по адаптации VidClaw под наши требования (по шагам, описано в диалоге):

1. **Модель задачи и поля**
   - Добавить в `tasks.json` и модели `project`, `orgComment`, `reviewComment`, `acStatus` (или `source`).
   - Обеспечить заполнение этих полей из API/UI при создании/синхронизировании задачи.

2. **Орг-агент и лимит параллельных задач**
   - Использовать существующее `settings.maxConcurrent` для лимита в 2 задачи + индикатор свободных слотов на UI.
   - При `pickup` сохранять `subagentId`, фиксы `startedAt`, `orgSession`, обновлять `runHistory`.
   - После отчета субагента поставить `status=needs_review` и логировать `result`/`error`.

3. **UI-карточки, статусы и комментарии**
   - Показывать `project`/`source` на карточке, добавить фильтрацию/тег.
   - Для колонки `needs_review` добавить кнопки: `Сделано` (перевод в `done`, добавление `result`) и `Переделать` (возврат в `inbox`, `pickedUp=false`, `subagentId=null`, коммент от Орга).
   - Отображать комментарии/отчеты субагента под описанием (из `result`, `error`, `runHistory`).

4. **История и комментарии**
   - Записывать `orgComment`, `reviewComment`, `runHistory` в `activity` для прозрачности.
   - При `Переделать` автоматически копировать последний `result` и добавлять новый комментарий.

5. **Интеграция issue tracker → VidClaw**
   - Написать webhook/скрипт `createya` → `/api/tasks` (с `project`, описание, `source=issue`), чтобы задачи попадали прямо в колонку "входящие".
   - Позволить создавать задачи вручную или через API (ты, Стив) без AC.

6. **Орг-ограничения и очереди**
   - Heartbeat/cron сам следит за `remainingSlots`. При освобождении слота automatically `pickup` следующую.
   - Кнопка/логика для возврата задачи в `inbox` (если `rework`).

7. **Документация и тесты**
   - Обновить `API.md`/документы, описать workflow и webhook.
   - Протестировать: create → pickup → subagent отчёт → review → done/rework → повтор.

---
Следующий шаг: начать выполнение первого пункта — расширить модель задачи и API, чтобы карточки знали о проекте и комментариях. После этого двигаться по порядку.  
Если надо, могу сразу подготовить PR с изменениями в `server/controllers/tasks.js` и фронте.
